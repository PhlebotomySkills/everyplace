# Ambient place-sound library

This folder holds seamless-looping MP3s for the "place sounds" audio mode. When a viewer toggles the music button to its second position on a Street View location, the site fetches `/sounds/{slug}.mp3` from this folder. If the file doesn't exist, nothing breaks, the mode just stays quiet.

Live YouTube cameras already have real place audio, so they don't need files here. The button unmutes the live stream directly.

## Filename convention

Each file is named after the location's slug. The slug is generated by lowercasing, stripping diacritics and punctuation, and joining words with hyphens. Examples:

| Location | Filename |
|---|---|
| Oia, Santorini | `oia.mp3` |
| Mount Fuji, Kawaguchi | `mount-fuji-kawaguchi.mp3` |
| Jemaa el-Fnaa | `jemaa-el-fnaa.mp3` |
| Sydney Opera House | `sydney-opera-house.mp3` |

To get the exact slug for any location, open the browser console on everyplace.live and run `LOCATIONS.map(l => l.slug)`.

## Audio specs

Target specs for a good loop:

- Format: MP3, 128kbps is enough (this is ambient background, not music)
- Length: 2 to 4 minutes is ideal. It will loop, but longer reduces the loop-detection fatigue.
- Volume: normalize to around -18 LUFS. Too loud and it fights the UI.
- Seamless: trim so the start and end meet cleanly. Audacity's crossfade-loop feature or the built-in "loop" test in most DAWs does this in 30 seconds.
- No copyright issues: source from freesound.org (filter for CC0 or Attribution 4.0) or record your own.

## Where to source audio (all free, all legal)

- [freesound.org](https://freesound.org) — huge library, filter by license. Search "ambient", "city", "beach", "wind", etc.
- [BBC Sound Effects](https://sound-effects.bbcrewind.co.uk/) — 33,000+ sounds, Personal/Educational license.
- [zapsplat.com](https://www.zapsplat.com/) — free with account, attribution not required for most.
- [xeno-canto.org](https://xeno-canto.org/) — bird sounds by species/region, if you want specific native birds at each place.

## Edge cases

If a sound file is detected as missing (404 or autoplay blocked), the music button stays in its "place sounds" visual state so the viewer knows what was intended. The fallback behavior is silence, not an error.
